OpenClaw Skills: A Complete Guide

By Sol AI · April 2026 · ~20 min read

If you're setting up OpenClaw for the first time, you'll quickly realise that the base install doesn't do much on its own. The agent needs to be taught what tools it has access to and how to use them. That's where Skills come in. They're essentially the thing that separates a basic AI chat interface from an agent that can actually manage your inbox, run web searches, or trigger automations on your behalf.

What is a Skill in OpenClaw?

A Skill is an extension that teaches your OpenClaw agent how to interact with a specific tool or service. Each one lives in its own folder and is built around a single file called SKILL.md — a plain-text document with a name, a description, and natural language instructions that tell the agent when and how to use it.

Writing a skill is closer to writing a clear brief for a colleague than it is to writing code. You describe what the tool does, when the agent should reach for it, and how to use it correctly. The agent figures out the rest.

Skill Scope and Priority

According to OpenClaw's official documentation, skills can come bundled with the software, be installed globally across all agents, or be scoped to a specific workspace. The priority order when two skills share the same name:

Priority Scope Location
1 (highest) Workspace-level skill Your workspace's skills/ folder
2 Globally installed skill ~/.openclaw/skills/
3 (lowest) Bundled default skill Shipped with OpenClaw

This means you can override any bundled skill with your own version — just drop a skill with the same name into your workspace. Useful for customising behaviour without forking the original.

What a SKILL.md Contains

A skill's SKILL.md file has three core sections:

  • Name and description — tells the agent what the skill does and when to consider it
  • Usage instructions — how to invoke it, what inputs it needs, what outputs to expect
  • Metadata — required environment variables, binaries, OS compatibility, and any external dependencies

Because skills are plain text, they're easy to inspect, fork, and modify. No compilation, no special tooling — just open the file and read what it actually does.

Where do Skills Come From?

Bundled Skills

OpenClaw ships with a baseline set of skills covering common tasks: web search, summarisation, browser automation. These are the ones you should start with. They're maintained by the core team, reviewed before shipping, and don't require trusting a third party.

ClawHub Marketplace

For anything beyond the basics, the main source is ClawHub, the official skill marketplace. You install skills from there using a single command:

clawhub install <skill-name>

The community index has grown fast. The Awesome OpenClaw Skills repository on GitHub catalogues over 5,400 skills as of early 2026, covering everything from Google Ads management to academic paper retrieval. The ecosystem is large — and as with any large ecosystem, quality varies significantly.

Roll Your Own

You can write your own. Because a skill is just a SKILL.md file, the barrier to entry is low. If you understand what a tool does and can describe it in plain language, you can teach OpenClaw to use it. No programming required — just clear thinking.

How Skills Work Under the Hood

When OpenClaw loads, it scans your skill directories and filters out any skills that can't run in your current environment. A skill that requires a specific API key or system binary won't load if those aren't present. The SKILL.md file carries metadata specifying those requirements.

Environment Filtering

Skills declare their requirements upfront. OpenClaw checks these before loading:

Requirement Type Example
Environment variables OPENAI_API_KEY, HIMALAYA_CONFIG_PATH
System binaries ffmpeg, python3, gh
Operating system darwin (macOS only), linux
npm packages @openclaw/skill-himalaya

If a requirement is missing, the skill is silently skipped. No errors, no crashes — just a skill that isn't loaded because the environment can't support it.

Multi-Agent Scoping

In multi-agent setups, you can scope a skill to a single agent via its workspace, or share it across all agents on the same machine through ~/.openclaw/skills. This lets you build agents with different skill profiles — one for research, one for communication, one for scheduling — running on the same machine without interference.

What You Can Do with OpenClaw Skills

Skills let you build out a personalised automation setup without writing a full application. Users have put together:

  • Email triage workflows — sort, flag, and respond to messages automatically
  • Flight check-in bots — navigate airline websites and handle check-in flows
  • Lead generation pipelines — scrape, enrich, and route prospect data
  • Calendar orchestration — coordinate across Google Calendar, Outlook, and Apple Calendar

The markdown-based format makes skills easy to inspect and customise. If a skill does 80% of what you need but has a rough edge, you can read the SKILL.md, understand it, and modify it. You don't need to wait for a maintainer to ship an update.

Routing Between Agents

For teams running multiple agents, you can route tasks between agents with different skill sets. One agent handles research, another handles communication, a third handles scheduling. OpenClaw's workspace scoping means each agent only sees the skills relevant to its role.

OpenClaw Skills vs Claude Skills

These sound similar but operate on fundamentally different models.

OpenClaw Skills Claude Skills
Platform Local-first, on your machine Cloud-based, tied to Claude.ai
What they control What the agent can execute How the model responds
Dependency model Carries own dependencies (API keys, binaries) Account-level saved instructions
Reasoning engine Pluggable: Claude, GPT-4o, DeepSeek, etc. Anthropic's Claude only
Access scope Your full machine — email, files, calendar Session context within Claude.ai

Where Claude's Skills shape how the model responds, OpenClaw Skills define what the agent can execute. This distinction matters: an OpenClaw skill isn't just a set of instructions — it can trigger real actions in the world.

The Security Risks

Warning: This section contains important information about real security risks with third-party OpenClaw skills. Read it before installing anything from the community marketplace.

The security concerns with OpenClaw skills aren't theoretical. Cisco's AI security research team tested a third-party OpenClaw skill and found it performed data exfiltration and prompt injection without the user's knowledge. The skill appeared to function normally while silently sending sensitive data elsewhere and manipulating the agent's instructions.

A contributor to the ecosystem estimated that roughly 80% of skills on ClawHub are either low quality or outright malicious. That's a striking number. It doesn't mean every skill is dangerous — but it means you should assume nothing and verify everything.

The Access Scope Problem

The deeper concern is access scope. OpenClaw agents can touch your email, calendar, messaging apps, and file system. A poorly written skill has all of that as potential blast radius. There are documented cases of agents deleting entire email inboxes during automated cleanup tasks — not maliciously, but because a skill's instructions weren't specific enough about what "cleanup" meant.

One of OpenClaw's own maintainers said the project is "far too dangerous" for anyone who can't run a command line safely. That's not a marketing line — it's a realistic assessment of what software that has access to your digital life can do if misconfigured.

ClawHub's Response

OpenClaw has since partnered with VirusTotal for security scanning. This helps catch known malware, but it's not a complete solution — a novel malicious skill can slip through signature-based detection. VirusTotal reports are a signal, not a guarantee.

What You Can Do About It

  1. Start with bundled skills before going anywhere near the community marketplace
  2. Read every SKILL.md before installing — if you can't understand what a skill does, don't install it
  3. Check the VirusTotal report — it's there for a reason
  4. Use sandboxed runs for skills that handle untrusted input or execute system commands
  5. Audit your installed skills regularly — remove anything you don't recognise or can't verify

The ecosystem is genuinely useful. The skills that exist cover an impressive range of real-world automation. But the security model requires you to be an active participant — not a passive consumer.

Getting Started

Finding What Skills Are Installed

# Check your skill directories
ls ~/.openclaw/skills/          # global skills
ls ~/.openclaw/workspace/skills/  # workspace skills

# List skills via OpenClaw CLI
openclaw skills list

Installing from ClawHub

# Install a skill by name
clawhub install <skill-name>

# Install to a specific workspace
clawhub install <skill-name> --workspace /path/to/workspace

# List available skills
clawhub search <keyword>

Writing Your Own Skill

A skill is just a folder with a SKILL.md file. Here's the minimal structure:

my-skill/
  SKILL.md
  scripts/
    do_the_thing.sh

And the SKILL.md might look like:

# My Tool Skill

Use this skill when you need to [describe the task].

## What it does
[Clear description of the tool's function]

## Requirements
- Binary: `my-tool` (install via `brew install my-tool`)
- Environment: `MY_TOOL_API_KEY` must be set

## Usage
Run the script:
```
./scripts/do_the_thing.sh --input <value>
```

## Output
Returns [describe output format]

That's it. No schema to learn, no config format to memorise. If you can explain it to a colleague, you can write a skill.

Enabling and Disabling Skills

# Disable a skill temporarily (rename the folder)
mv ~/.openclaw/skills/skill-name ~/.openclaw/skills/skill-name.disabled

# Re-enable
mv ~/.openclaw/skills/skill-name.disabled ~/.openclaw/skills/skill-name

Verifying a Skill Before Installing

  1. Read the full SKILL.md
  2. Check what binaries or scripts it references
  3. Look at the VirusTotal report if one exists
  4. Check the GitHub repository activity (last commit? open issues?)
  5. Search for the skill name + "security" or "review"
