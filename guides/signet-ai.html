Signet-AI Integration Guide for OpenClaw

A comprehensive guide to installing, configuring, and using Signet-AI as a persistent memory layer for OpenClaw.

What is Signet-AI? An open protocol for agent identity, knowledge, and trust. It provides ambient memory extraction and persistence across sessions, tools, and environments — without replacing your existing harness.

Overview

Why Signet-AI?

Most AI agents only remember when explicitly told to. That’s not memory — it’s a filing cabinet.

Signet makes memory ambient. It extracts and injects context automatically between sessions, before the next prompt starts.

Key benefits: - Less prompt re-explaining between sessions - One memory layer across Claude Code, OpenCode, OpenClaw, and Codex - Clear visibility into what was recalled, why, and from which scope - Local-first storage (your data stays on your machine)

Benchmark Performance

On the LoCoMo benchmark (standard for conversational memory systems):

Rank System Score
1 Kumiho 97.5%
6 SLM V3 Mode C 87.7%
7 Signet 87.5%
8 Zep/Graphiti ~85%

Signet ranks 7th overall and is the highest-ranked local-first solution (no LLM at search time).

Prerequisites

  • macOS, Linux, or WSL
  • Bun or npm
  • OpenClaw installed and running
  • 500MB free disk space

Check your environment:

bun --version   # or: npm --version
openclaw --version

Installation

Step 1: Install Signet Globally

Using Bun (recommended):

bun add -g signetai

Or using npm:

npm install -g signetai

Verify the installation:

signet --version
# Should output: 0.86.4 (or higher)

Step 2: Run the Setup Wizard

Launch the interactive setup:

signet setup

Non-interactive setup (for automation/scripting):

signet setup \
  --non-interactive \
  --path /Users/amre/.openclaw/workspace \
  --name "Your Agent Name" \
  --description "Your agent description" \
  --deployment-type local \
  --network-mode localhost \
  --harness openclaw \
  --embedding-provider native \
  --extraction-provider none \
  --openclaw-runtime-path plugin \
  --configure-openclaw-workspace \
  --skip-git \
  --allow-unprotected-workspace

Step 3: Link Your OpenClaw Workspace

If Signet isn’t automatically linked to your OpenClaw workspace:

# Check current workspace status
signet workspace status

# Set workspace to your OpenClaw directory
signet workspace set /Users/amre/.openclaw/workspace

Step 4: Restart the Daemon

signet daemon restart

OpenClaw Configuration

Signet integrates with OpenClaw via runtime plugins. The setup wizard should automatically patch your OpenClaw configuration, but here’s what it does:

What Gets Configured

The Signet setup creates/updates:

  1. agent.yaml — Agent manifest in your workspace root
  2. signet runtime — Registers Signet as a memory provider
  3. OpenClaw hooks — Session-memory integration

Manual Configuration (if needed)

Your OpenClaw config is at ~/.openclaw/openclaw.json. The Signet setup adds a session-memory hook:

"hooks": {
  "internal": {
    "entries": {
      "session-memory": {
        "enabled": true
      }
    }
  }
}

Verification & Testing

1. Check the Dashboard

signet dashboard

Opens http://localhost:3850 — view memories, retrieval, and system health.

2. Test Memory Recall

Save a test memory:

signet remember "Testing Signet persistence - March 2026"

Recall it:

signet recall "March 2026 testing"

3. Test Cross-Session Continuity

In one session:

signet remember "My favorite color is blue"

In a new session (or after restarting OpenClaw):

signet recall "favorite color"

You should see the memory returned without re-injection.

Usage

CLI Commands

Command Description
signet status Show daemon and memory status
signet doctor Run health checks
signet dashboard Open web UI
signet remember "fact" Save a memory
signet recall "query" Search memories
signet logs View daemon logs
signet daemon restart Restart the daemon

Memory Search

# Basic recall
signet recall "project details"

# JSON output (for scripting)
signet recall "project details" --json

# With filters
signet recall "emails from march" --scope work

Managing Memories

# Edit a memory
signet memory edit <memory-id>

# Delete a memory  
signet memory delete <memory-id>

# Export all memories
signet export --output ./signet-backup.json

Architecture Analysis

How Signet Works

┌─────────────────────────────────────────────────────┐
│                    Your Harness                      │
│              (OpenClaw, Claude Code, etc.)          │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│              Signet Runtime Plugin                   │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐ │
│  │  Extraction │  │   Storage   │  │   Recall    │ │
│  │   Pipeline  │─▶│   Layer     │◀─│   Engine    │ │
│  └─────────────┘  └─────────────┘  └─────────────┘ │
│         │                │                │         │
│         ▼                ▼                ▼         │
│  ┌─────────────────────────────────────────────────┐│
│  │          SQLite + Markdown (Local)              ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘

Key Components

  1. Extraction Pipeline — Distills session content into structured memories
  2. Storage Layer — SQLite for relational data, Markdown for transcripts
  3. Recall Engine — Hybrid search (vector + keyword) with provenance tracking
  4. Dashboard — Web UI for inspecting memory and retrieval

Data Storage

Data Type Storage Location
Memories SQLite memory/memories.db
Transcripts Markdown memory/transcripts/
Config YAML agent.yaml
Identity Markdown AGENTS.md, SOUL.md, etc.

All data is local-first — nothing leaves your machine by default.

Search Architecture

Signet uses hybrid search: - Vector search (nomic-embed-text) for semantic similarity - Keyword search (BM25/FTS5) for exact matches - Reranking to optimize results

Alpha parameter (0-1) controls the blend: - alpha: 0.7 (default) — lean toward semantic - alpha: 0.3 — lean toward keyword

Troubleshooting

Daemon Not Running

# Start the daemon
signet daemon start

# Check status
signet status

Missing Identity Files

If you see Missing identity files: AGENTS.md, SOUL.md, IDENTITY.md, USER.md:

# Verify workspace path
signet workspace status

# Reset to correct workspace
signet workspace set /path/to/your/workspace
signet daemon restart

OpenClaw Not Integrating

# Re-run setup with OpenClaw harness
signet setup --harness openclaw --configure-openclaw-workspace

# Restart daemon
signet daemon restart

Memory Recall Not Working

# Check search index
signet embed audit

# Rebuild index if needed
signet embed backfill --force

Dashboard Won’t Load

# Check if port 3850 is in use
lsof -i :3850

# Restart dashboard
signet dashboard

Advanced Configuration

Changing the Workspace Path

signet workspace set /new/path --force
signet daemon restart

Switching Embedding Provider

Default is native (nomic-embed-text). Alternatives:

# OpenAI
signet config --embedding-provider openai --embedding-model text-embedding-3-small

# Ollama (local)
signet config --embedding-provider ollama --embedding-model llama3

Search Tuning

Adjust the alpha parameter for search balance:

signet config --search-balance 0.5

Lower alpha = more keyword-focused. Higher alpha = more semantic.

Memory Decay

Configure how aggressively old memories are deprioritized:

# In agent.yaml
memory:
  decay_rate: 0.95  # Higher = faster decay
  session_budget: 2000  # Max sessions to retain

Multi-Agent Setup

Signet supports isolated/shared memory for multiple agents:

# Add a named agent
signet agent add sol --description "Primary assistant"

# Switch between agents
signet agent use sol
signet agent use dev

Security Notes

  • Local-first: Data never leaves your machine unless you explicitly sync
  • Encrypted secrets: Agent-blind secret storage available
  • Inspectable: All memories can be viewed, edited, deleted
  • No vendor lock-in: Export your entire memory graph as JSON
